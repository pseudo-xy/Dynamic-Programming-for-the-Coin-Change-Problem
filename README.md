# Dynamic-Programming-for-the-Coin-Change-Problem
This project applies dynamic programming to solve the classic coin change problem: given a target amount of 91 cents (change for $1.09 from $2.00), find the minimum number of coins needed to make exact change. 

The project is structured in three parts, each building on the previous to draw broader conclusions about coin systems and algorithmic approaches.
Part 1 solves the problem using standard US coin denominations (1¢, 5¢, 10¢, 25¢, 50¢). The DP formulation defines f(s) as the minimum number of coins needed to make s cents, with f(0) = 0 as the base case. For each amount from 1 to 91, the algorithm iterates over all available coins and updates f(s) if using that coin leads to fewer total coins than previously found. A traceback array records which coin was last used at each amount, enabling reconstruction of the exact coin breakdown.
Part 2 discusses the theoretical relationship between dynamic programming and the greedy algorithm for coin change. It explains that standard US coins form a canonical system — each denomination is structured such that always picking the largest coin that fits never leads to a suboptimal solution. For canonical coin systems, greedy and DP produce identical results, and DP serves as a formal verification of this property.
Part 3 introduces a non-standard 20¢ coin alongside the original denominations and re-solves the problem. Non-canonical coin systems can cause the greedy algorithm to fail — for example, greedy might pick a 25¢ coin when using a 20¢ coin would ultimately require fewer total coins. The DP approach handles this correctly regardless of coin structure.
A final comparison table summarizes the results of both coin sets, showing whether adding the 20¢ coin reduces the number of coins needed for 91¢ and highlighting cases where the expanded denomination set offers an advantage. The full DP table for each scenario is printed, showing the optimal value and coin used at every amount from 0 to 91.
